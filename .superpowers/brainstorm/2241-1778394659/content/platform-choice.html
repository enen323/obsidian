Obsidian — 软件实操教程站

选技术栈: 网站类型?

项目 Obsidian — 教程网站 + 联盟营销。哪种技术栈合适?

A

Java Spring Boot (动态站点)

匹配现有 Maven 项目。Spring Boot + Thymeleaf/JSP + MySQL。完整后端,用户系统,管理后台。

Pros

  • 全栈控制
  • 用户系统内置
  • 成熟生态

Cons

  • 开发慢
  • 部署复杂
  • 教程站过重
B

静态站点 (HTML/CSS/JS + SSG)

Hugo/Jekyll/纯静态。Markdown 写教程,生成 HTML。轻量,快,便宜。

Pros

  • 极速加载
  • 免费托管(GitHub Pages)
  • 开发简单

Cons

  • 无后端
  • 无用户系统
  • 管理需手动
C

Next.js / JAMStack

React 驱动的 SSG + SSR 混合。Markdown/MDX 内容,可选后端 API。

Pros

  • 现代开发体验
  • SEO 优秀
  • Vercel 免费托管

Cons

  • React 学习曲线
  • Node.js 环境
  • 教程站可能过杀
D

WordPress

传统 CMS。插件生态丰富,联盟营销插件现成。可视化编辑。

Pros

  • 上手最快
  • 联盟插件多
  • 管理方便

Cons

  • 臃肿
  • 安全需维护
  • 定制受限
