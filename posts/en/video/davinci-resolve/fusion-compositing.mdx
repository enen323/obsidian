## Why Does Your Composite Look "Fake"?

Visual compositing is where skill gaps show most clearly. Green spill around keyed edges, floating elements from tracking drift, mismatched lighting -- these issues were called "the compositor's nightmare triad" in a 2024 NAB survey, with 92% of professional compositors admitting they faced all three early in their careers.

Unlike After Effects' layer-based compositing, Fusion uses a **node-based** workflow. The core advantage: every processing step is an independent module you can inspect and modify at any point without affecting others. For complex composites (10+ elements), node-based workflows are roughly 3x more efficient than layer-based.

This tutorial walks you through a complete visual composite project in DaVinci Resolve's Fusion page: green screen keying, background replacement, color matching, motion tracking, and final output.

## Core Workflow

### Step 1: Enter the Fusion Page

Open your project in DaVinci Resolve. Place both the green screen clip and background on the timeline (Edit page) -- green screen on V1, background on V2 aligned with the clip.

Switch to the **Fusion** page (click the Fusion icon in the bottom toolbar, or press **Shift+5**).

**Expected result**: The center of the screen shows the **Nodes Editor** (initially empty). Media Pool is on the left, Inspector and Spline/Keyframes panels on the right. The selected timeline clip automatically generates a MediaIn node.

### Step 2: Build the Basic Node Tree

The fundamental node structure in Fusion is: `MediaIn → Processing → MediaOut`.

Entering the Fusion page, your selected clip automatically has **MediaIn** and **MediaOut** nodes. Press **Tab** to quickly search and add nodes.

Press **Tab** (or Shift+Space) to open the node search box, type `Background`, select the Background node, and click. Drag the output (small gray triangle on the right) of the Background node to connect to the input (small gray triangle on the left) of MediaOut.

> **Why add the background first**: In Fusion, the connection order determines the processing flow. Wrong connections can result in a black screen or non-functional effects.

**Expected result**: A simple connection in the Node Editor -- MediaIn and Background both feeding into MediaOut.

### Step 3: Chroma Keying (Green Screen)

Press **Tab** and search for **Delta Keyer**. Place it between MediaIn and MediaOut: connect MediaIn's output to Delta Keyer's input, and Delta Keyer's output to MediaOut's input.

Select the Delta Keyer node, then in the Inspector panel on the right:

1. Click the **Color Picker** (eyedropper icon)
2. Click the green area of the green screen in the Viewer -- DaVinci Resolve automatically analyzes the green range and performs the key
3. If one click isn't enough, hold Shift and click additional green areas

Adjust keying parameters in the Inspector:
| Parameter | Suggested Value | Description |
|-----------|-----------------|-------------|
| Tolerance | 10-30 | How wide a range of green to select |
| Softness | 5-15 | Edge feathering to reduce aliasing |
| Gamma | 0.5-0.8 | Controls semi-transparent edge transitions |
| Black/White Clip | Default 0 | Extreme edge cleanup, rarely needed |

**How to check key quality**: In the Viewer dropdown menu, select **Alpha** mode (shortcut **A**). In Alpha view: black = fully transparent, white = fully opaque, gray = semi-transparent.

- Edges should show smooth gray transitions (hair detail, semi-transparent areas)
- The person's interior should be pure white (opaque)
- Background should be pure black (transparent)

**Why Delta Keyer over Color Keyer**: Delta Keyer is optimized specifically for green/blue screen keying in Fusion, using color difference algorithms (Delta = difference) to separate foreground from background. It handles semi-transparent areas (hair, glass) far better than other keyers.

**Stop condition**: In Alpha mode, person edges are smooth with no visible green spill or jagged edges. In Color mode, the background area is transparent.

### Step 4: Edge Refinement

Keyed edges often have **green spill** -- green light reflected onto the subject's edges. Add a **Spill Supressor** node:

1. Press Tab, search for Spill Supressor, add it after Delta Keyer
2. In the Inspector, set **Color To Suppress** to **Green** (default)
3. Adjust **Suppression** -- start at 50%, increase until green spill disappears
4. **Range** controls the effect's reach -- 0.5 to 0.8 works well

> **Why green spill happens**: Green screen light reflects onto the subject, especially hair and shoulder edges. Spill Supressor analyzes color differences to remove these reflections -- far more effective than simply desaturating green.

Next, add an **Erode/Dilate** node (Tab search "erode"):
- In the Inspector's Filter tab, set Erode/Dilate to **1.0** (slight edge erosion to remove residual green borders)
- Set **Blend** to **0.3-0.5** to keep the effect natural

**Expected result**: No green halo around the person -- clean, natural edges.

### Step 5: Composite the Background

Ensure background media feeds in through a Background node or another MediaIn node. For a still image background:

1. Add a **Loader** node (Tab search "loader"), select your background image file
2. Or use the background clip already on the Edit page timeline (appears as another MediaIn node)

Connect the Loader/MediaIn output to the **Merge** node's **Background** input (yellow input port, marked with a yellow square). Connect Delta Keyer's output to Merge's **Foreground** input (green port). Connect Merge's output to MediaOut.

**Adjust the composite**:
- Select the **Merge** node. In the Inspector, the default blend mode is **Over** -- correct for green screen compositing.
- If dimensions don't match, add a **Transform** node after the background's MediaIn or Loader, adjusting Size (scale) and X/Y (position)
- Position and scale the background for a natural composition

**Expected result**: The Viewer shows the person composited onto the background with clean edges and a balanced composition.

### Step 6: Color Matching

The foreground and background almost certainly have mismatched lighting and color tones. Use a **Color Correct** node for matching:

1. Add a **Color Correct** node between Delta Keyer and Merge (on the foreground line)
2. In the Inspector, adjust:
   - **Lift** (shadows): Warm if the background is warm, cool if it's cool
   - **Gamma** (midtones): Match the overall midtone color
   - **Gain** (highlights): Adjust highlight color tones
   - **Brightness**: Overall brightness matching

**Faster approach**: Use a **Color Curve** node:
1. Add a Color Curve node, switch the Viewer to **Split** mode (top/bottom split showing before/after)
2. Fine-tune RGB channels with curves so the foreground blends into the background

**Why color matching matters**: Lighting in a green screen environment almost never matches the background scene perfectly. Even with perfect keying, color mismatch makes the composite look fake. The human visual system is especially sensitive to skin tones -- a 5% hue shift is enough for viewers to sense something is wrong.

**Stop condition**: Foreground and background share consistent color tones. Skin looks natural with no green or blue cast.

### Step 7: Motion Tracking

If the camera moves (handheld or gimbal), the background needs to follow the foreground's motion.

1. Add a **Tracker** node after the foreground's MediaIn (Tab search "track")
2. In the Inspector, set the tracker mode to **Translation**
3. In the Viewer, drag the tracking point to a high-contrast area on the foreground (a logo on a collar, near an eye)
4. Click **Track Forward** (the ▶ button) to auto-track
5. After tracking completes, click **Create Keyframes** or **Apply**

**Tracker mode selection**:
| Mode | Best For | Description |
|------|----------|-------------|
| Translation | Pure pan/tilt | Simplest, fastest -- tracks X/Y position only |
| Rotation | Translation + rotation | Use when the subject rotates |
| Scale | Translation + scale | Camera moves forward/backward |
| Affine | Translation + rotation + scale + skew | General handheld camera motion |
| Perspective | Full perspective tracking | Most accurate but most compute-intensive |

**Expected result**: During playback, the background tracks the foreground movement stably with no visible "sliding" or "jitter."

### Step 8: Export the Composite

Switch to the **Deliver** page (or export from the Edit page).

**Export settings**:
| Parameter | Value |
|-----------|-------|
| Format | QuickTime |
| Codec | H.264 or ProRes 422 |
| Resolution | Match project resolution |
| Frame Rate | Match project |
| Quality | Best |

**Stop condition**: The exported video plays correctly with composite quality maintained across all frames.

## Troubleshooting

### 1. Green Halo Around Keyed Edges

**Symptom**: A green glow surrounds the subject's edges, especially noticeable in hair.

**Root cause**: Green screen light reflected onto the subject's edges (green spill). Delta Keyer didn't fully remove the reflected light.

**Fix**:
1. In Delta Keyer's Inspector, increase **Softness** to 10-20
2. Add a **Spill Supressor** node, raise Suppression to 60-80%
3. Use **Erode/Dilate** with a positive value (1.0-2.0) to slightly erode the edge
4. If persistent, adjust **Alpha Gamma** in Delta Keyer to 1.2-1.5

### 2. Tracking Drift / Background Jitter

**Symptom**: During playback, the background slides or jitters relative to the foreground.

**Root cause**: The tracking point was placed on a low-contrast area, or the tracking mode was too simple (e.g., Translation for a rotating scene).

**Fix**:
1. Select a higher-contrast tracking region (ideally 100x100 px with visible texture)
2. If the foreground rotates, switch to Rotation or Affine tracking mode
3. After tracking, manually inspect keyframes and correct tracking points where jitter occurs
4. Use **Planar Tracker** (Fusion Studio's advanced tracker) for surface tracking

### 3. Semi-Transparent Areas (Hair/Smoke) Fail to Key

**Symptom**: Hair strands or transparent objects (glass, smoke) are either fully removed or badly fragmented by Delta Keyer.

**Root cause**: Delta Keyer defaults toward hard edges, struggling with semi-transparent areas.

**Fix**:
1. Lower **Gamma** in Delta Keyer to 0.3-0.5 to preserve more semi-transparent detail
2. Enable **Clean Black/White** and reduce thresholds
3. Add a **MatteControl** node, adjusting **Blur** and **Choke** in the Inspector to smooth the matte
4. For complex scenes, use Resolve Studio 18.5+'s **Magic Mask** (AI-based masking) -- significantly more accurate than traditional chroma keying for difficult subjects

### 4. Fusion Page Lag / Slow Preview

**Symptom**: Every parameter adjustment in Fusion takes seconds to update in the Viewer.

**Root cause**: Complex node trees stress GPU and memory resources. The free version also lacks GPU acceleration support.

**Fix**:
1. Right-click nodes not needing real-time adjustment and select **Bypass**
2. Use **Cache to Disk** (right-click menu) to cache rendered results
3. In the Fusion page Viewer settings (top right), reduce display resolution to **Half** or **Quarter**
4. Consider upgrading to the Studio version for GPU acceleration

### 5. Exported Composite Has Color Shift

**Symptom**: The composite looks correct in Fusion but appears washed out or too dark when played in a media player after export.

**Root cause**: Color space mismatch. Fusion works in a linear color space by default, but export likely needs Rec.709 Gamma 2.4.

**Fix**: Add a **ColorGamut** node before MediaOut with:
- Input Color Space: **DaVinci YRGB** or **Linear**
- Output Color Space: **Rec.709 Gamma 2.4**
- Or set the project color science to **DaVinci YRGB Color Managed** so Resolve handles color transforms automatically

## Expert Tips

### Compositing Efficiency Data

From a survey of 200 professional compositors (FMX 2024):

- **Node vs. layer workflow**: Compositors using node-based workflows averaged 4.2 hours for complex composites (15+ elements), compared to 11.5 hours for layer-based workflows. **2.7x efficiency gain**.
- **Green screen keying time**: A professional compositor completes a standard green screen key (with background matching) in 12-18 minutes, including detail refinement. Beginners typically take 45-60 minutes.
- **AI-assisted masking**: Compositors using Magic Mask and Neural Engine features achieved a 94% success rate on hair and transparent objects, vs. ~67% for manual keying.

### Hardware Optimization Tips

1. **Fusion cache settings**: In DaVinci Resolve Preferences > Fusion, set Fusion Memory Limit to 70% of physical RAM (e.g., 22 GB for 32 GB systems). Enable **Disk Cache** and select a fast SSD as the cache drive.

2. **Proxy mode**: In the Fusion page Viewer's top toolbar, set **Proxy Mode** to **Auto** or **Half Resolution**. Previewing complex composites (especially 4K+) at full resolution creates unnecessary performance overhead.

3. **Node management**: Use **Pipe Routers** (right-click in Node Editor > Add Tool > Pipe Router) to organize connection lines and reduce visual clutter. Color-code nodes using right-click > Set Color (blue = keying, green = color, orange = tracking) for faster visual parsing.
