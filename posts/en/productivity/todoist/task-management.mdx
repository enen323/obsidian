## Why Your To-Do List Keeps Falling Apart

Most people use a to-do app the same way: Day one, they enthusiastically add every task they can think of. Three days later, they stop opening the app. One week later, they open it to find dozens of overdue tasks — feel defeated, and quit.

The problem is not your willpower. The problem is that your to-do list has been designed as a "wish list" rather than an "execution system." Any task management system depends on trust — your trust that a task entered into the system will surface at the right time. When more than 30 tasks pile up in Today view, that trust collapses instantly.

Todoist's superpower is its filter and priority system — you do not need to dump everything into one list. Learn to use projects, labels, priorities, and filters correctly, and your to-do list will transform from a stress source into a stress reliever.

This tutorial covers Todoist's five core functions: task creation, project management, labels and filters, priorities, and habit tracking.

## Core Workflow

### Step 1: Create Tasks Quickly

Todoist supports multiple creation methods. These are the three fastest:

1. **Quick Add bar**: Click the input field at the top of any view (shortcut: `Q`), type your task, and press Enter.
2. **Natural language dates**: Include date phrases in your task — Todoist parses them automatically. Examples:
   - "Submit report **today**" → Due today
   - "Dentist appointment **tomorrow at 3pm**" → Due tomorrow 15:00
   - "Buy gift **Friday**" → Due this Friday
   - "Return books **every 1st**" → Recurring monthly task
3. **Mobile quick actions**: iOS Shortcuts and Android widgets let you add tasks from the notification shade without opening the app.
4. **Why**: Manually selecting a date (click calendar → pick date → confirm) takes 10–15 seconds on average. Natural language takes 2 seconds. Creating 5–10 tasks daily, natural language saves 20–45 minutes per month.
5. **Stop when**: You can create tasks on both desktop and mobile, and you use natural language dates fluently.

### Step 2: Structure Your Projects

Projects are the top-level organizational unit in Todoist. The principle is "fewer is better."

1. Click "Add Project" in the left sidebar (or press `Shift+P`) and create this structure:
   ```
   📁 Work
   📁 Personal
   📁 Learning
   📁 Archive
   ```
2. Add sections inside the "Work" project:
   - Click the project → Add section: `Development`, `Meetings`, `Documentation`
3. **Why**: Keeping projects between 4–6 is optimal. More than 8 causes decision fatigue — each time you add a task, you spend energy on "which project" instead of "what needs to be done." Sections are lighter than nested projects and are recommended for within-project organization.
4. **Expected result**: The sidebar shows 4 main projects with sections under each. Tasks are organized by section within projects.
5. **Stop when**: You have a clean 4–6 project structure with 2–3 sections per project.

### Step 3: Use Labels and Filters (Pro Feature)

Labels are Todoist's most powerful metadata system. They cut across projects, enabling cross-project views.

1. Create common labels in Settings → Labels:
   ```
   @important    — High priority but not urgent
   @waiting      — Blocked, waiting for someone else
   @quick        — Can be done in under 15 minutes
   @phone        — Requires a phone call
   ```
2. Apply labels to tasks: Click the label icon in task detail, or type `@labelname` when adding a task.
3. Create filters under the Filters view:
   - **Today's focus**: `today & p1`
   - **Waiting on**: `@waiting & !assigned to others`
   - **Quick wins**: `@quick & !today`
4. **Why**: Projects group by "domain" (Work/Personal/Learning); labels group by "action type" (Important/Waiting/Quick). A task like "Call client to confirm requirements" belongs under the Work project, but it can also carry `@phone` and `@waiting` labels. Filters let you view by action type instead of by domain.
5. **Stop when**: You have created at least 4 labels and set up at least 1 filter that surfaces a useful cross-project task list.

### Step 4: Master Priority Levels

Todoist has four priority levels (p1–p4), color-coded. Using them correctly is more practical than simply "marking important tasks."

1. **p1 (red)**: Must be completed today. No more than 3 per day.
2. **p2 (orange)**: Important tasks for this week.
3. **p3 (blue)**: Tasks for this month.
4. **p4 (no color)**: No time pressure — get to it when you can.
5. **Usage**: Select a task and press `P+number` (`P1`, `P2`, `P3`, `P4`) to set priority quickly.
6. **Why**: Relative priority (p1–p4) beats simple "important/unimportant" because it counters the cognitive bias that "everything is urgent." A hard rule: if Today has more than 3 p1 tasks, re-evaluate — you cannot meaningfully complete 4 "most important" things in one day.
7. **Stop when**: Your Today view has no more than 3 p1 tasks, sorted p1 → p2 → p3.

### Step 5: Build Habit Tracking

Todoist supports habit tracking through recurring tasks — but the setup matters.

1. Create a habit task:
   - Title: `Morning meditation 10 min`
   - Recurrence: `every day`
   - Priority: `p3` (habits do not need high priority)
   - Project: `Personal`
2. Advanced recurrence patterns:
   - `every weekday` — Monday to Friday
   - `every 1st` — Monthly on the 1st
   - `every 3 days` — Every three days
   - `every Monday, Wednesday, Friday` — Specific days
3. Review progress: Check completed tasks in Today view. Todoist tracks consecutive completion streaks.
4. **Why**: The secret to habit tracking is not "reminders push you" — it is "seeing a streak makes you not want to break it." Todoist's recurring tasks leverage the "don't break the chain" psychology: once you complete 7 consecutive days, skipping day 8 carries a noticeable psychological cost.
5. **Stop when**: You have created 3 habit tasks (at least one daily and one weekly) and see them auto-populate in Today view.

## Troubleshooting

1. **Symptom: "Natural language date was not recognized"** → **Root cause**: Todoist's language setting does not match your input language. **Fix**: Go to Settings → Language and set it to match your writing language. For English, set to "English."

2. **Symptom: "Labels do not appear in the label list"** → **Root cause**: The free plan does not include labels. **Fix**: Labels are a Pro feature ($4/month). On the free plan, use sections as a limited substitute for label-like categorization.

3. **Symptom: "Completed recurring task disappeared from history"** → **Root cause**: Editing the content of a completed recurring task reopens the completed instance instead of creating the next one. **Fix**: Never edit the content of an already-completed recurring task — create a new one. To change future recurrence rules, edit the "Every" field (this applies the change starting from the next instance).

4. **Symptom: "No push notification received on mobile"** → **Root cause**: System notification permissions are not granted, or Todoist's notification settings are off. **Fix**: Grant Todoist notification permission in iOS/Android system settings. In Todoist Settings → Notifications, ensure "Task reminders" is enabled.

5. **Symptom: "Sidebar has too many projects and scrolls slowly"** → **Root cause**: Project count exceeds 10 with no archiving. **Fix**: Right-click projects inactive for 6+ months → "Archive project." Archived projects stay searchable but do not appear in the sidebar. This keeps the sidebar clean without data loss.

## Expert Tips

- **The 2-minute rule in practice**: Any task that can be completed in under 2 minutes should not enter Todoist at all — do it immediately. This sounds counterintuitive (a task manager telling you not to enter tasks?), but it is a core GTD principle. A 2-minute task costs at least 30 seconds of overhead to create and check off — not worth it.
- **Inbox zero every morning**: Spend 5 minutes at the start of each workday processing your inbox — assign project, date, and priority to each task. When inbox exceeds 20 unprocessed tasks, system trust erodes. Keep it under 10.
- **Use Todoist templates (Pro)**: Save common project structures as templates (Project menu → "Save as template"). Create a template for "New client project" with sections for kickoff, research, deliverables. Each new project saves 15–20 minutes of setup by applying the template.
- **Board view for stage-based projects**: Toggle to Board view (top-right of any project) to display tasks as cards in columns by section. This is more intuitive than a flat list for projects with clear stages like "To Do → In Progress → Review → Done."
