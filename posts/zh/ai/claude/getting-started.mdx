## Claude 和 ChatGPT 到底有什么不一样？

也许你用过 ChatGPT，觉得 AI 对话工具就是那样——打出问题，收到答案。但每次处理长篇文档时，看到"内容超出长度限制"的提示，你只能把文章切成几段分开问，结果上下文接不上，AI 的回答前言不搭后语。

Claude 的核心差异在于：**超长上下文窗口和处理深度**。

Claude 原生支持 200K token 的上下文窗口（约 15 万英文单词或 15 万中文字符）。你不需要拆分文件，直接丢进去一整个文档，Claude 可以完整理解和分析。这改变了你使用 AI 的方式——从"分段投喂"变成"整本阅读"。

本文从零开始，带你熟悉 Claude 的核心功能。

## 核心工作流：Claude 五大核心功能

### 第 1 步：基础对话——学会提问

Claude 的对话框和 ChatGPT 类似，但有几点关键差异：

**最佳提问方式**：

```
请分析以下项目计划的潜在风险。先列出所有可能的风险点，然后按影响程度从高到低排序，最后给出每个风险的缓解方案。

项目计划：[粘贴或上传文件]
```

**Claude 的回复特点**：
- 偏好结构化输出（列表、表格、标题层级）
- 回复较 ChatGPT 更详细和全面
- 对事实性错误的纠正请求响应很好（说"你刚才说的 X 不太准确，因为……"即可）

**为什么这样提问**：同时要求了三种输出（列出、排序、给出方案），Claude 的结构化输出能力强，一步到位得到完整分析。

### 第 2 步：文件上传——发挥硬件级优势

Claude 的文件上传支持比大多数竞品更强。支持的格式：

| 格式 | 最大文件 | 典型用途 |
|------|---------|----------|
| PDF | ~100MB | 报告、论文、合同 |
| TXT | ~10MB | 代码、笔记 |
| CSV | ~50MB | 数据表格 |
| 图片 (JPG/PNG) | ~20MB | 图表截图、扫描件 |
| Markdown | ~10MB | 文档 |

**上传技巧**：直接把文件拖入对话框。Claude 会自动读取内容，你可以立即开始提问——不需要等文件"处理"。

```
（上传一个 50 页的 PDF 报告后）
请总结这份文档的要点：
1. 核心结论（3 句话内）
2. 关键数据（表格形式）
3. 作者的主要论证逻辑
4. 你发现的数据矛盾或逻辑漏洞
```

**预期效果**：Claude 能在 10 秒内读完整个文档并给出结构化摘要——一个人类需要 2-3 小时才能读完的量。

### 第 3 步：长文处理——200K 上下文实战

这是 Claude 的杀手级功能。你可以把整本书、整个代码库或一整年的报告一次性交给 Claude。

```
请完整阅读以下代码库。然后：
1. 画出架构图（用 ASCII 或 Mermaid 格式）
2. 识别设计模式
3. 指出最可能出 Bug 的 3 个模块并说明原因
4. 给出重构建议

[粘贴整个代码库的关键文件]
```

**为什么有效**：传统 AI 只能看到几页代码，Claude 可以看到整个代码库，跨文件的调用关系一目了然。

**注意事项**：
- 虽然上下文窗口很大，但在超长对话中（超过 50 轮），Claude 对早期内容的记忆会减弱
- 关键内容放在对话的后半部分或使用"记住刚才提到的……"来强化

### 第 4 步：编程辅助——代码生成与审查

Claude 的代码能力在 GPT-4 级别，优势在于**一次性处理的代码量**。

```
审查以下 Python 代码。请：
1. 找出所有潜在 Bug（标注行号）
2. 识别性能瓶颈
3. 检查类型安全
4. 给出优化后的完整代码

代码：
[粘贴代码]
```

**关键提示词技巧**：在编程任务中，加上"使用最新的语言特性"或"兼容 Python 3.12+"可以引导 Claude 使用较新的语法。

### 第 5 步：写作协作——从构思到终稿

Claude 在长文写作方面表现突出，特别适合论文、报告、技术文档。

**完整写作工作流**（三步，建议分三个对话完成）：

```
对话 1 - 大纲阶段：
我想写一篇关于 Serverless 架构的文章。请生成 3 个不同的文章大纲，风格分别是：技术深度型、商业决策型、入门普及型。

对话 2 - 写作阶段（使用确认的大纲）：
按照我们确认的大纲 2（商业决策型）开始写作。保持每个章节 300-400 字。语言正式但不过于学术。

对话 3 - 精修阶段：
优化上面文章的以下方面：
1. 第一段的钩子不够有力，请重写
2. 第三部分的论证缺少数据支撑，请补充
3. 检查全文的一致性——术语使用是否统一
```

## 常见问题排查

### Q1: 上传的 PDF 内容无法被读取

**原因**：PDF 是扫描件（图片格式），不是文本格式。Claude 目前不支持 OCR。
**解决**：使用 OCR 工具（如 Adobe Acrobat 的 OCR 功能）将扫描件转为可搜索 PDF 后再上传。或者上传图片格式（JPG/PNG）并描述图中的内容。

### Q2: 对话变长后回答质量下降

**原因**：尽管 Claude 有 200K 上下文窗口，但在 50+ 轮对话后，注意力分布会偏向近期内容。
**解决**：对关键任务启用新对话。在长对话中定期使用"请总结一下我们目前讨论的关键点"来刷新上下文。或者把前期结论手动粘贴到新对话中。

### Q3: 代码审查的结果不够深入

**原因**：没有给 Claude 具体的审查维度。泛泛的"审查这段代码"得到泛泛的回答。
**解决**：指定审查维度并给出代码上下文。使用"审查代码，重点关注安全漏洞（特别是 SQL 注入和 XSS）、性能瓶颈（循环和数据库查询）、以及 TypeScript 类型错误"。

### Q4: Claude 的回答有时过于冗长

**原因**：Claude 的设计偏好是输出详细的回答（这是其风格特点）。
**解决**：在提示词中明确指定输出长度。例如"200 字以内"、"用 5 个要点概括"、"只给代码，不要解释"。

## 专家技巧

- **使用 Mermaid 图表**：Claude 原生支持生成 Mermaid 格式的流程图、时序图和甘特图。直接要求"用 Mermaid 画出这个流程"即可得到可渲染的图表代码。
- **项目功能**：Claude 的 Projects 功能（独立文章）允许将多份文档和自定义指令绑定到一个项目空间中。非常适合"写作项目"或"代码库维护"场景。
- **分阶段写作**：对于长文，不要在一轮对话中完成"生成提纲→写全文→校对"。分三轮对话完成，每轮结束后把结果复制出来，开启新对话继续。质量提升约 30%。
- **知识截止日期**：Claude 的训练数据截止时间取决于版本。当你需要最新信息时，可以开启 Claude 的联网搜索功能（需手动点击启用）。
- **对话标题管理**：Claude 自动生成对话标题，但你可以手动修改。给重要对话加上描述性标题（如"电商项目-数据建模讨论-0525"），方便后续检索。
