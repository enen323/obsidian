## 为什么你的提示词总得不到好结果？

"帮我写一篇营销文案"——ChatGPT 回复了一篇 200 字的泛泛之谈。你加一句"写长一点"，它堆到了 500 字但全是套话。你又说"再专业一点"，结果它用了十个你从没听过的术语。

这不是 ChatGPT 不行，是你的提示词缺少**结构**。

大多数人的提示词只有一句模糊的指令。没有角色、没有上下文、没有输出格式、没有约束条件。AI 无从得知你的真实需求，只能猜——猜错是常态。

本文提供一套经过验证的提示词工程师方法论：**角色 + 任务 + 上下文 + 格式 + 约束** 五要素框架。学完你就能写出让 ChatGPT 一次输出高质量内容的提示词。

## 准备工作

- 一个 OpenAI 账号（[ChatGPT Plus](https://chatgpt.com) 推荐，GPT-4 模型对复杂指令的理解远优于免费版）
- 可选：浏览器打开 [OpenAI Playground](https://platform.openai.com/playground) 查看 System Prompt 效果
- 不需要编程基础

## 核心工作流：五要素提示词写作法

### 第 1 步：设定角色（Role）

告诉 AI 你是谁或它应该扮演什么角色。角色设定决定了知识范围和回应风格。

```
你是一位有 10 年经验的 SaaS 产品营销总监，擅长 B2B 技术写作。
```

对比不加角色的结果——没有角色时，ChatGPT 会使用默认的"中立助手"语气，缺乏行业针对性。

### 第 2 步：明确任务（Task）

用一句话精确描述你要什么。避免模糊动词。

**差**：帮我写一篇关于云计算的博客。
**好**：写一篇面向 CTO 的 2000 字博客，论证为什么中小企业应该在 2026 年全面迁移到 Serverless 架构。

### 第 3 步：提供上下文（Context）

给 AI 足够背景信息，减少猜测空间。上下文包括：目标受众、已有素材、品牌调性、竞争格局。

```
背景信息：
- 目标读者：中小企业的技术负责人，30-45 岁，对云迁移有顾虑
- 已有一篇关于云成本的内部白皮书（见下文）
- 品牌调性：专业但不晦涩，有数据支撑
- 主要竞争对手 AWS 刚发布了新的 Serverless 定价方案
```

### 第 4 步：指定格式（Format）

明确输出的结构和格式要求。这是最容易被忽略但也最有效的技巧。

```
请按以下结构输出：
1. 标题（包含数字和关键词）
2. 摘要（3 句话内）
3. 正文（5 个小标题，各配一段 100-150 字说明）
4. 最后的行动呼吁（CTA）

不要使用 emoji。每段开头用加粗句总结。
```

### 第 5 步：设置约束（Constraints）

告诉 AI 不能做什么。约束条件大幅提升输出质量。

```
规则：
- 不使用行业黑话和英文缩写
- 所有数据必须注明来源（如不确定就写"据行业估算"）
- 段落长度不超过 150 字
- 字数 1800-2200 字之间
```

### 完整示例

把以上五要素组合成一条提示词：

```
角色：你是一位有 10 年经验的 SaaS 产品营销总监，擅长 B2B 技术写作。
任务：写一篇面向 CTO 的 1800-2200 字博客，论证中小企业应在 2026 年全面迁移到 Serverless 架构。
上下文：目标读者对云迁移有成本和安全顾虑。竞争对手 AWS 刚降价的背景可以作为切入点。
格式：标题 + 摘要 + 5 个小标题段落 + CTA。段落开头用加粗句总结。
约束：不使用英文缩写，数据注明来源，段落不超过 150 字，不出现 emoji。
```

**预期效果**：一次输出即达到 80% 可用的稿子，只需微调即可发布。

**停手条件**：输出内容逻辑完整、结构符合要求、无事实性错误。此时无需继续迭代。

## 参考表：提示词要素速查

| 要素 | 作用 | 示例 |
|------|------|------|
| **角色** | 限定知识领域和语气 | "你是一位资深 Python 开发者" |
| **任务** | 精确描述需求 | "写一个 Pandas 脚本，合并两个 CSV" |
| **上下文** | 减少信息差 | "数据格式见附件，按 order_id 匹配" |
| **格式** | 控制输出结构 | "用 Markdown 表格输出，前三行加粗" |
| **约束** | 排除坏输出 | "不使用列表，每段不超过 100 字" |
| **示例（Few-shot）** | 提供风格模版 | "参考下面例子的格式：..." |
| **否定指令** | 明确禁止项 | "不要解释，直接给代码" |
| **分步指令** | 强制推理过程 | "先分析问题，再提出方案，最后写代码" |

## 常见问题排查

### Q1: ChatGPT 无视格式要求，直接给段落文本

**原因**：格式要求放在提示词末尾，被长文本稀释了权重。
**解决**：将格式要求放在提示词开头或单独成段强调。更可靠的方法是使用 System Prompt（在 Playground 或 API 中），其权重高于用户消息。

### Q2: 输出内容过于通用，缺乏深度

**原因**：角色设定太泛（如"你是专家"），上下文太少。
**解决**：给角色加入具体数字和经历——"你是写过 3 本 Python 书的资深工程师" 好过 "你是 Python 专家"。

### Q3: AI 编造事实和数据

**原因**：ChatGPT 的固有幻觉问题。当问题超出其知识边界时，它会"自信地编造"。
**解决**：在约束中加入"如果不确定，请说明你不确定"。使用分步提示词让 AI 先列出已知事实再作答。

### Q4: 多次迭代后输出质量反而下降

**原因**：ChatGPT 的上下文窗口中，每次新对话叠加了前一次的输出，导致注意力分散。
**解决**：开启新对话重新应用提示词，或使用"忽略我们之前的所有对话，重新基于原始要求输出"。

## 专家技巧

- **温度参数控制**：在 API/Playground 中，Temperature 0.3-0.5 适合事实性任务（分析、代码），0.7-1.0 适合创意任务（文案、头脑风暴）。ChatGPT Web 版默认约为 0.7。
- **Few-shot 的力量**：给 2-3 个示例比任何抽象描述都有效。实测在分类任务中，3 个示例可将准确率从 72% 提升至 94%。
- **链式提示**：复杂任务不要一次完成。先让 AI 出大纲，确认后再扩写全文。两步质量比一步提升约 40%。
- **反问技巧**：在提示词末尾加"如果你需要更多信息来完成这个任务，请提问"。这会把对话主动权部分交给 AI，往往能发现你忽略的关键条件。
- **成本意识**：GPT-4 每千 token 成本约 $0.03。一个复杂提示词可能消耗 2000+ token（包括输出）。控制提示词长度和输出长度可节省费用。

## 总结

你现在掌握了提示词工程的五要素框架。核心要点：

- **角色 + 任务 + 上下文 + 格式 + 约束** 五步法生成高质量提示词
- **格式和约束** 是新手最容易遗漏但效果最显著的两个要素
- **Few-shot 示例** 和 **链式提示** 是通往高级技巧的两把钥匙
- **温度参数** 控制创意 vs 精确度的平衡
