> **适用版本：** Microsoft 365（Power Query 内置于 Excel 2016+ 和 Office 365 订阅）  
> **难度：** ★★★★☆（4/5）  
> **预计用时：** 50-65 分钟

---

每周你都会收到来自 ERP 系统的相同 CSV 导出文件。列标题大小写不一致。日期有三种不同的格式。有些行多了列，把后续数据全部挤偏。每个周一的前两个小时，你都在修复同样的问题。

电子表格式的做法——手动查找替换、复制列、删除重复项——缓慢、易错且无法追溯。下周你还得重来一遍。

Power Query（官方名称"获取和转换数据"）是 Excel 内置的 ETL 引擎。它会记录你执行的每一个转换步骤，并在新数据上以一键方式回放。无需 VBA，无需手动返工。本教程教会你完整的 Power Query 工作流：连接数据源、清洗列、合并和追加查询、自动化整个流程。

---

## 准备工作

- **软件：** Microsoft 365 桌面版（Excel 2016+ 也包含 Power Query，但 365 版提供最多连接器）。Excel 2016/2019 用户请确认已启用 Power Query 插件（数据选项卡 > 获取数据）。
- **操作系统：** 推荐 Windows 10/11。macOS 上的 Power Query 连接器较少（无 Active Directory、无 Exchange、较少文件源）。
- **示例文件：** 准备至少两个结构相同但数据不同的 CSV 文件（如 sales-1月.csv、sales-2月.csv）。有意让数据凌乱些：不统一的表头、混合的日期格式、多余空格、空行。
- **前置知识：** 熟悉 Excel 表格和基本公式。无需 Power Query 或 M 语言经验。
- **关键思维转变：** Power Query 是**只读**的——它从不修改原始数据。每次转换都创建新的数据集。你随时可以回退或更改任何步骤。

## 核心工作流

### 第一步：将数据导入 Power Query

**为什么重要：** Power Query 的连接引擎可以从文件、数据库、Web API 等读取数据。导入步骤定义了数据源。

**导入 CSV 文件：**
1. 转到 **数据 > 获取数据 > 来自文件 > 来自文本/CSV**。
2. 选择你的 CSV 文件，点击导入。
3. 在预览对话框中，确认 Power Query 正确检测了分隔符（逗号、制表符等）和数据类型。
4. 点击 **转换数据**（不是加载）进入 Power Query 编辑器。

**导入命名的 Excel 表格：**
1. 选中源表格中的任一单元格。
2. 转到 **数据 > 来自表格/区域**。
3. Power Query 打开并载入了你的表格。

**预期结果：** Power Query 编辑器窗口打开。右侧的**查询设置**窗格显示一个步骤："源"。左侧预览你的数据。

**停止条件：** 你可以在 Power Query 编辑器中看到数据，所有原始行列完整保留。

### 第二步：清洗列——删除、重命名、更改类型

**为什么重要：** 源数据很少具有正确的列结构。清洗可以在分析之前消除噪音并规范类型。

**删除列：** 选中列（Ctrl+点击多选）> 开始 > 删除列 > 删除列。或右键列标题 > 删除。

**重命名列：** 双击列标题，输入新名称。或右键 > 重命名。

**更改数据类型：** 点击列标题旁的数据类型图标（数字为"123"，文本为"ABC"，日期为日历图标）。选择正确的类型。

**常见转换：**

| 操作 | 方法 |
|---|---|
| 删除空行 | 开始 > 删除行 > 删除空行 |
| 删除重复项 | 选中列 > 开始 > 删除行 > 删除重复项 |
| 替换值 | 右键单元格 > 替换值 |
| 向下填充 | 右键列 > 填充 > 向下 |
| 透视/逆透视列 | 选中列 > 转换 > 透视列 或 逆透视列 |

**示例——清洗"全名"列：**
1. 按分隔符拆分：右键"全名"列 > 拆分列 > 按分隔符 > 空格 > 确定。
2. 修剪空格：选中两个新列 > 转换 > 格式 > 修整。
3. 重命名："全名.1"改为"姓氏"，"全名.2"改为"名字"。

**预期结果：** 只保留需要的列，名称正确，数据类型得当。

**停止条件：** 数据具有清晰的列标题、正确的类型，且没有不需要的行。

### 第三步：合并查询——按键组合表

**为什么重要：** VLOOKUP 的 Power Query 版本？合并查询（Merge）做同样的事，但更快、更透明、且不会在数据变化时崩坏。

**场景：** 你有 `销售表`（订单ID、产品ID、数量、金额）和 `产品表`（产品ID、产品名、类别）。需要将产品名合并到销售表中。

1. 在 Power Query 编辑器中，选中 **销售表** 查询（左侧窗格）。
2. 转到 **开始 > 合并查询**。
3. 在对话框中：
   - 选择 **产品表** 作为第二个表。
   - 点击两个表中的 **产品ID** 列建立匹配。
   - 保留默认联接种类：**左外部**（销售表的所有行，产品表的匹配行）。
   - 点击确定。
4. 在销售表数据中，出现一个新列 `产品表`，其中包含 Table 值。
5. 点击 `产品表` 列标题上的展开图标（双箭头）。
6. 取消勾选"使用原始列名作为前缀"。只勾选**产品名**和**类别**。
7. 点击确定。

**预期结果：** 销售表查询中出现了产品名和类别列，通过产品ID关联。

**停止条件：** 你已合并两个表，验证主表中的每一行都具有查找所得的值。

### 第四步：追加查询——纵向堆叠表格

**为什么重要：** 每月报表来自不同文件。追加查询将多个表合并为一个长表，用于汇总视图。

**场景：** 你有 `销售-1月`、`销售-2月` 和 `销售-3月` 三个查询（各自从独立的 CSV 导入）。

1. 在 Power Query 编辑器中，转到 **开始 > 追加查询**。
2. 选择**三个或更多表**。
3. 将 `销售-1月`、`销售-2月`、`销售-3月` 从左框移到右框。
4. 点击确定。

**替代方法——从文件夹追加：**
1. 转到 **数据 > 获取数据 > 来自文件 > 来自文件夹**。
2. 选择包含所有 CSV 文件的文件夹。
3. 点击**合并 > 合并和转换数据**。
4. Power Query 自动导入所有文件，检测共同架构，并追加数据。
5. 如果架构不同，使用"示例文件"参数指定模板。

**预期结果：** 一个查询包含来自所有月度文件的所有行。"已应用步骤"列表显示：源、...、追加的查询。

**停止条件：** 你可以将新月份的文件放入文件夹，刷新查询，合并后的表格自动更新。

### 第五步：透视和逆透视列

**为什么重要：** 逆透视将宽格式的汇总表转变为高格式的分析就绪表。透视则反向操作。

**逆透视（最常见——修复"坏"的交叉表数据）：**

你有一个这样的表：
| 产品 | Q1 | Q2 | Q3 | Q4 |
|---|---|---|---|---|
| 甲 | 100 | 120 | 110 | 130 |

你希望：
| 产品 | 季度 | 销售额 |
|---|---|---|
| 甲 | Q1 | 100 |

1. 选中 **产品** 列。
2. 转到 **转换 > 逆透视其他列**。
3. 将"属性"重命名为"季度"，将"值"重命名为"销售额"。

**透视（使高格式数据变宽）：**
1. 选中 **季度** 列。
2. 转到 **转换 > 透视列**。
3. 值列：选中 **销售额**。
4. 在高级选项下：选择**求和**作为聚合方式。
5. 点击确定。

**预期结果：** 数据从宽格式变为高格式（逆透视）或从高格式变为宽格式（透视）。通过检查行数变化来验证。

**停止条件：** 你可以将交叉表报告转换为适合透视图表或进一步分析的扁平表格。

### 第六步：应用、刷新和自动化

**为什么重要：** Power Query 的全部意义在于可重用性。一旦设置好，你只需刷新，无需重做。

**将结果加载到 Excel：**
1. 转到 **开始 > 关闭并加载**（如需选择目标位置，则选"关闭并加载到"）。
2. 选择**表**（默认）或**数据透视表**。
3. 选择**新工作表**或**现有工作表**。

**源数据变化时刷新：**
- 右键结果表 > **刷新**。
- 或转到 **数据 > 全部刷新**（快捷键：Ctrl+Alt+F5）。
- 所有查询都会刷新。已应用步骤从头到尾回放。

**打开文件时自动刷新：**
1. 转到 **数据 > 查询和连接**。
2. 右键查询 > **属性**。
3. 勾选**打开文件时刷新数据**。

**预期结果：** 一个实时更新的 Excel 表，一个刷新命令即可回放所有转换步骤。

**停止条件：** 你可以添加新的源文件、更改源数据或修复数据格式问题，刷新查询即可看到清洗后的结果——无需触碰转换步骤。

## 故障排查

### 1. "找不到文件"错误
- **现象：** 将 Excel 文件移到其它文件夹后，Power Query 显示"找不到文件"。
- **原因：** 查询的源步骤存储了绝对文件路径。移动文件后链接断裂。
- **解决：** 转到数据 > 查询和连接 > 右键查询 > 属性。在"定义"下点击"浏览"重新指向新位置。对于便携式工作簿，将 CSV 放在相对文件夹路径中，使用**数据 > 获取数据 > 来自文件 > 来自文件夹**替代。

### 2. 刷新后数据类型错误
- **现象：** 刷新后某列从文本变为数字（或反之），导致工作表中的公式出错。
- **原因：** Power Query 的"检测数据类型"设为每次自动检测。新的数据批次若有混合类型，Power Query 可能会猜错。
- **解决：** 在已应用步骤列表中删除"更改的类型"步骤，手动显式设置每列的数据类型。然后刷新——Power Query 会尝试强制转换，错误会显示为"Error"，提醒你有问题行。

### 3. 合并结果出现大量空值
- **现象：** 合并后，展开列的大多数行显示为"null"。
- **原因：** 连接键列不精确匹配——多余空格、不同大小写或不同的数据类型。
- **解决：** 合并前清洗两个键列：使用转换 > 格式 > 修整处理空格，确保两者数据类型一致（同为文本或同为数字）。某些情况下，使用自定义列 `= Text.Trim([键列])`。

### 4. 大文件性能缓慢
- **现象：** 处理 50 万行以上文件时，Power Query 编辑器响应需要数分钟。
- **原因：** Power Query 将完整数据加载到内存用于预览。每一步都重新计算预览。
- **解决：** 在查询早期应用筛选，减少行数。在查询设置 > 属性中，对重量级查询取消"允许后台下载"。对于生产级 ETL，考虑使用 Power BI 或其他专用数据工具。

### 5. 追加查询后列错位
- **现象：** 追加两个表后，列没有对齐——数据显示在错误的列中。
- **原因：** 两个表中的列名不同（一个用"客户名称"，另一个用"客户名"）。Power Query 按列名匹配，而非位置。
- **解决：** 在追加前标准化所有源文件的列名。在 Power Query 中重命名每个查询的列，使它们完全一致。或使用"从文件夹合并文件"方法，它能自动检测并对其架构。

## 专家技巧

**实际时间节省：** 一家中型公司的财务主管用 Power Query 自动化了每月合并流程。原本需要 8 小时的手工数据组装（打开 12 个部门的文件、复制粘贴、重新格式化），现在只需 3 分钟——一键刷新查询链。

**查询依赖关系很重要：** 在 Power Query 编辑器中，转到**视图 > 查询依赖关系**。这显示查询之间如何相互馈送。如果查询 A 合并到查询 B，查询 B 再合并到查询 C，那么刷新 C 会刷新下游的所有内容。用这个来规划你的查询顺序。

**动态筛选的参数查询：** 为日期范围或产品筛选创建一个参数（右键查询窗格 > 新建参数）。在"筛选行"步骤中引用这个参数。更改参数值并刷新；整个查询链动态筛选。无需手动编辑步骤。

**从示例创建列：** 在"转换"选项卡中，"从示例创建列"让你在新列中输入想要的结果，Power Query 会推断出 M 代码。例如：如果你在 01/15/2024 旁边输入"2024-01-15"，Power Query 会自动检测日期转换公式。这是学习 M 的最快方式。

**性能监控：** Power Query 编辑器在底部状态栏显示步骤执行时间。如果一个步骤用时超过 5 秒，考虑优化：将筛选前移、移除不必要的列、或对重复查找使用 Table.Buffer()。

**错误处理：** 数据类型更改后，某些单元格显示"Error"。转到开始 > 删除行 > 删除错误。这将完全删除问题行。若要先调查，添加一个自定义列，内容为 `= try [列] otherwise "检查"`，然后查看哪些行有问题。

- **透视与逆透视**——在宽格式和高格式之间重塑
- **刷新与自动化**——一键数据更新
