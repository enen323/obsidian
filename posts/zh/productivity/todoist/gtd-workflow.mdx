> **适用版本**：Todoist 2026（需 Pro 订阅解锁标签和筛选器，这是 GTD 工作流的关键支撑）
> **难度等级**：★★☆☆☆（需熟悉 Todoist 基本操作，已阅读[任务管理入门](/zh/productivity/todoist/task-management)）
> **预计耗时**：初始搭建 45 分钟，每周回顾维护 15 分钟

## 为什么"把事情做完"这么难

你可能读过 David Allen 的《Getting Things Done》——收集、处理、组织、回顾、执行，五步流程看起来逻辑严密。你在笔记本上画过工作流图，在 Notion 里搭过 GTD 数据库——然后呢？两周后回到了原来的状态：脑子里塞满了待办事项，每天晚上睡前列"明天要做的事"，第二天又被新的事情淹没。

GTD 的崩溃点从来不是方法论本身有问题——而是它要求的"每周回顾"和"收件箱处理"需要一套趁手的工具来降低操作成本。用纸笔做 GTD，处理一条"等快递"就需要写卡片、贴标签、归档——操作成本太高。用 Todoist 做 GTD，一条任务从收集到归档只需要 10 秒。

本教程将教你如何用 Todoist 完整实现 GTD 五步流程，包括具体的项目结构、标签体系、筛选器配置和每周回顾模板。

## 准备工作

- **Todoist Pro 订阅**：GTD 工作流依赖标签、筛选器和自定义提醒，这些是 Pro 功能（$4/月）。
- **已完成**：[Todoist 任务管理入门](/zh/productivity/todoist/task-management)——确保你已掌握任务创建、项目、标签和优先级的基本操作。
- **时间预算**：初始搭建 45 分钟（一次性），之后每天 5 分钟处理收件箱 + 每周 15 分钟回顾。
- **（可选）阅读**：David Allen 的《Getting Things Done》——理解方法论原理有助于你在遇到边界情况时做出正确的决策。

## 核心工作流

### 第一步：设置 GTD 项目结构

GTD 将任务分为几种类型——你需要对应的项目来承载它们。

1. 在 Todoist 中创建以下项目：
   ```
   📁 收件箱          — GTD 的"收集篮"，所有新想法先丢到这里
   📁 下一步行动       — 所有已明确的下一个物理行动
   📁 等待中          — 需要别人推进的任务
   📁 将来/也许       — 想做但不是现在
   📁 参考资料        — 没有行动价值但有信息价值的条目
   📁 项目清单        — 需要多个步骤才能完成的任务集合
   ```
2. 在"项目清单"下按当前实际项目添加分区，例如：
   ```
   📁 项目清单
       ├── 网站改版
       ├── 年终汇报
       └── 装修计划
   ```
3. 将默认的"收件箱"重命名为 GTD 的"收集箱"概念——所有新任务的第一站。
4. **为什么这么做**：GTD 的核心原则是"大脑不应该用来存储任务，而应该用来执行任务"。这 6 个项目对应 GTD 的 6 个分类——所有任务在你系统中都只能属于其中一类。没有"不知道放哪里"的中间地带。
5. **停止条件**：你的 Todoist 左侧栏显示这 6 个 GTD 项目，并且你理解每个项目的用途。

### 第二步：创建 GTD 标签体系

GTD 的"上下文"概念——你在什么场景下执行任务——通过 Todoist 标签来实现。

1. 在设置 → 标签中创建以下上下文标签：
   ```
   @电脑      — 需要在电脑前完成
   @外出      — 需要出门办理（银行、邮局等）
   @电话      — 需要打电话
   @会议      — 需要面对面沟通
   @思考      — 需要深度思考和规划（不涉及具体操作）
   ```
2. 创建能量标签：
   ```
   @高能量    — 需要集中注意力
   @低能量    — 5-15 分钟的零碎任务
   ```
3. 创建时间标签：
   ```
   @<15分钟   — 快速任务
   @15-30分钟 — 中等任务
   @>30分钟   — 大块时间任务
   ```
4. **为什么这么做**：传统的 GTD 上下文标签（@电脑、@外出）解决的是"我在这个场景下能做什么"。能量标签和时间标签进一步优化——当你只有 15 分钟休息时间时，你会希望看到"低能量且小于 15 分钟"的任务，而不是整个"下一步行动"列表。
5. **停止条件**：你创建了至少 3 个上下文标签和 2 个能量标签。

### 第三步：从收集到处理——收件箱清零流程

这是 GTD 中最关键也最容易被忽略的步骤。

1. **收集阶段（全天）**：任何新任务、想法、承诺——立刻输入到"收件箱"。不要判断它重不重要、放哪个项目——先收集再说。快捷键 `Q` 快速添加。
2. **处理阶段（每天开始前 5 分钟）**：
   - 打开"收件箱"项目。
   - 逐条问自己两个问题：
     - **"这是可以执行的任务吗？"**
       - 否 → 移入"参考资料"（如果以后可能有价值）或"将来/也许"（如果以后想做），或直接删除。
       - 是 → 继续下一个问题。
     - **"完成它需要几步？"**
       - 一步 → 添加上下文标签、预估时间标签，设定日期（如果需要），移入"下一步行动"。
       - 多步 → 在"项目清单"中创建一个新项目，将第一个步骤放入"下一步行动"。
3. **为什么这么做**：收件箱是 GTD 的"门面"——它的状态直接影响整个系统的信任度。每天处理收件箱一次（而不是实时处理），因为实时处理会让你不断中断当前的工作流。
4. **停止条件**：你的收件箱每天结束时保持少于 5 条未处理任务。

### 第四步：设置 GTD 筛选器

筛选器让你按上下文查看任务，而不用翻阅整个项目列表。

1. 创建以下筛选器：
   ```
   📋 电脑前工作     : @电脑 & !p4
   📋 出门待办       : @外出 & !today
   📋 电话清单       : @电话 & !p4
   📋 快速任务       : @<15分钟 & (@高能量 | @低能量)
   📋 等待检查       : #等待中
   📋 将来/也许       : #将来/也许
   ```
2. 筛选器语法说明：
   - `@标签名` — 匹配标签
   - `#项目名` — 匹配项目（注意是 `#` 不是 `@`）
   - `&` — 与条件（同时满足）
   - `|` — 或条件（满足其中之一）
   - `!` — 排除（不要包含）
   - `p1/p2/p3/p4` — 优先级
   - `today / !today` — 日期条件
3. **为什么这么做**：筛选器是 GTD 工作流的核心引擎——"电脑前工作"筛选器让你坐下打开电脑后，立刻看到所有需要在电脑上完成的任务，而不需要先想"我在哪个项目"。
4. **停止条件**：你创建了至少 4 个筛选器，并能在筛选器视图中看到按上下文组织的任务列表。

### 第五步：每周回顾——GTD 的节拍器

每周回顾是 GTD 中唯一不能省略的步骤。它让系统保持可信。

1. **时间**：每周五下午 4:00–4:15（设置为 Todoist 中的重复任务：`每周五下午4:00`，p2）。
2. **回顾清单**：
   - [ ] 收件箱清零（处理到少于 3 条）
   - [ ] 检查"将来/也许"——有没有需要转正的？
   - [ ] 检查"项目清单"——每个项目是否都有下一步行动？
   - [ ] 检查"等待中"——跟催过期的任务
   - [ ] 检查"下一步行动"——是否有超过两周没动的任务？
   - [ ] 归档"参考资料"中不再需要的信息
3. 将这份清单保存为 Todoist 模板，或者直接在 Todoist 中以任务列表形式放在"项目清单"中。
4. **为什么这么做**：没有每周回顾，GTD 系统会在 2–3 周内崩溃。"等待中"的任务会被人遗忘，"将来/也许"会积压，"下一步行动"会包含过期的僵尸任务。每周 15 分钟的回顾是让系统保持信任的固定投资。
5. **停止条件**：你连续进行了 3 次每周回顾（这意味着你已经建立了这个习惯的雏形）。

### 第六步：自动化规则（Pro 高级功能）

Todoist Pro 支持自动化规则，可以自动处理重复性的任务管理操作。

1. 打开设置 → 规则，创建以下自动化：
   - **规则 1：每日任务准备**
     - 触发条件：每天 08:00
     - 执行动作：将所有"收件箱"中带 `@今天` 的任务移入"下一步行动"
   - **规则 2：标记过期任务**
     - 触发条件：任务到达截止日期
     - 执行动作：自动设置为 p2
   - **规则 3：周报提醒**
     - 触发条件：每周五 15:45
     - 执行动作：创建一个任务"进行每周回顾"，p1，放在"收件箱"中
2. **为什么这么做**：自动化规则处理那些你每次都做但容易忘记的操作。最有价值的是"每日任务准备"——你不需要在每天早晨手动把收件箱任务分类到"下一步行动"。
3. **停止条件**：你创建了至少 2 条自动化规则并观察它们正确执行。

## 常见问题排查

1. **症状：收件箱里的任务超过 50 条，完全不想处理** → **根源**：每天处理变成了每周/每月处理，积压导致心理负担。**解决**：先花 15 分钟做一次"大扫除"——不要逐条精处理，直接批量操作：超过 30 天未处理的直接归档（右键 → 归档），剩下的在 3 天内每天处理 10 条清零。

2. **症状：不记得"将来/也许"里有什么** → **根源**：没有定期审查"将来/也许"项目。**解决**：在每周回顾中增加"将来/也许审核"步骤。超过 3 个月没有动过的"将来/也许"任务，考虑删除——如果真的事后想起来，重新创建比面对一堆陈旧条目更有生产力。

3. **症状：项目清单里的项目没有下一步行动** → **根源**：GTD 的基本规则被违反——每个项目必须有一个"下一步行动"。**解决**：在每周回顾中，对每个"项目清单"中的项目确认其"下一步行动"字段。如果某项目没有下一步行动，问自己："我还在做这个项目吗？"→ 否，移入"将来/也许"或归档。

4. **症状：筛选器返回了预期之外的结果** → **根源**：筛选器语法错误，或标签/项目名称有空格或特殊字符。**解决**：检查语法——项目名用 `#` 而非 `@`；多个条件用 `&` 连接。如果项目名包含空格，用引号括起来：`#"我的项目"`。在筛选器测试模式下逐步添加条件来定位问题。

5. **症状：自动化规则没有触发** → **根源**：规则的触发条件设置错误，或 Todoist 客户端版本不支持自动化。**解决**：确认规则中的时间格式正确（24 小时制）。自动化规则需要 Todoist v1060+（2026 年版本），在 Web 端和桌面端运行，移动端暂时不支持。

## 专家技巧

- **"下一步行动"的评估指标**：一个健康的 GTD 系统，"下一步行动"项目中的任务数量应该在 20–40 条之间。低于 20 条说明你的 backlog 太干净（可能遗漏了任务），高于 40 条说明你收集得多、执行得少——需要减少收集或增加执行时间。
- **处理"接了一个球"的场景**：当别人通过聊天工具或邮件给你分配任务时：转发给 Todoist（在 macOS/iOS 上使用系统共享菜单 → Todoist，或使用 Todoist 的邮件转发功能 `add@todoist.com`）。目标是让 Todoist 成为你的唯一"收件箱"，而不是分散在微信、邮件、钉钉和 Teams 四个地方。
- **"将来/也许"的季度清理**：每个季度（3/6/9/12 月）安排一个 30 分钟的"将来/也许"审查。那些超过一个季度没有被关注过的条目，要么提升为"下一步行动"要么删除。这个季度节奏比每周回顾更宏观，确保你的"梦想清单"不会变成"遗忘清单"。
- **GTD + 番茄工作法结合**：用筛选器选出今天的 p1 和 p2 任务，然后用番茄钟执行。一个番茄钟（25 分钟）开始时，从筛选器中选一个任务专注执行。筛选器保证了你在番茄钟内不需要做"接下来做什么"的决定。

## 总结

你在 Todoist 中完整搭建了 GTD 工作流：6 个 GTD 项目（收件箱、下一步行动、等待中、将来/也许、参考资料、项目清单）、三层标签体系（上下文 + 能量 + 时间）、按上下文聚合的筛选器、每天 5 分钟的收件箱处理流程、每周 15 分钟的回顾，以及自动化规则。

GTD 的核心不在于方法论本身，而在于你信任这个系统。花一个月时间严格遵守这套流程——到第二个月，你会发现"大脑清空"的感觉比你想象的要好得多。
