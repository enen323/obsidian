> **适用版本**：Obsidian 1.8.x（Publish 需 v1.6.0+） | **难度等级**：★★★☆☆（需基本 Git 和命令行操作）
> **预计耗时**：Obsidian Publish 方案 15 分钟，GitHub Pages 方案 60 分钟

## 为什么分享笔记比写笔记更难

你的 Obsidian 知识库里可能已经有了几百篇笔记——它们相互链接、有完整的数据视图、是你数月积累的思考结晶。但如果有人问你要某篇笔记，你只能：截图发给对方、复制为纯文本粘贴到微信里、或者把整个 `.md` 文件发过去——链接断裂、格式全丢、图谱消失。

问题在于：Obsidian 的 `.md` 文件是给人读的，不是给浏览器读的。直接打开一个 Markdown 文件，链接不可点击、标签无法导航、图片路径错误。你需要一个"渲染中间层"将笔记转换为可访问的网站。

解决方案有三个层次：官方 Obsidian Publish（最快、付费）、Digital Garden 插件（免费、功能适中）、GitHub Pages + 静态站点生成器（最灵活、免费、需技术配置）。本教程覆盖三种方案，你可以根据预算和技术水平选择。

## 准备工作

### Obsidian Publish 方案
- Obsidian 账号（免费注册 [obsidian.md](https://obsidian.md)）
- Obsidian Publish 订阅（每月 $10 起）
- Obsidian 桌面端 v1.6.0+

### GitHub Pages + 静态站点方案（本教程以 Quartz 为例）
- GitHub 账号（免费注册 [github.com](https://github.com)）
- 安装了 Git（Windows: `winget install git` / macOS: `brew install git`）
- 安装了 Node.js 18+（[nodejs.org](https://nodejs.org) 下载 LTS 版本）
- 基本的命令行经验

### Digital Garden 插件方案
- Obsidian 桌面端
- GitHub 账号

## 核心工作流

### 方案一：使用 Obsidian Publish（5 分钟发布）

1. 在 Obsidian 中打开设置 → 第三方插件 → Obsidian Publish（核心插件，默认包含在安装包中，不需要从社区市场安装）。如果未启用，在"核心插件"列表中启用它。
2. 点击左侧功能区的地球图标（发布按钮）。
3. 使用 Obsidian 账号登录（在弹出的浏览器窗口中完成登录）。
4. 点击"选择一个站点"→"创建新站点"，输入站点名称（例如 `my-digital-garden`）。
5. 选择要发布的笔记——默认所有笔记都被勾选。你可以单独取消勾选某些笔记来控制可见性。
6. 点击"发布"按钮，Obsidian 会上传所选笔记的 Markdown 文件并渲染为网页。
7. 访问 `https://your-site-name.obsidian.md` 查看效果。
8. **为什么这么做**：Publish 保留了 Obsidian 生态中的所有特性——双向链接可点击、图谱视图可交互、反向链接可导航——这些是其他发布方案难以实现的。
9. **停止条件**：你的浏览器能打开 publish 站点，点击 `[[wikilink]]` 能跳转到目标笔记。

### 方案二：使用 Quartz + GitHub Pages（免费方案，45 分钟）

Quartz 是一个专门为 Obsidian 打造的静态站点生成器，能将整个知识库转换为响应式网站。

1. **Fork Quartz 模板**：打开浏览器访问 https://github.com/jackyzha0/quartz ，点击 "Use this template" → "Create a new repository"，命名为 `your-github-username.github.io`。
2. **克隆到本地**：开终端运行 `git clone https://github.com/your-username/your-github-username.github.io.git`。
3. **安装依赖**：在项目目录中运行 `npm install`。
4. **配置 Quartz**：编辑 `quartz.config.ts` 文件，修改 `pageTitle`、`description`、`baseUrl` 等字段：
   ```typescript
   const config: QuartzConfig = {
     configuration: {
       pageTitle: "My Digital Garden",
       baseUrl: "your-github-username.github.io",
       // ...
     }
   };
   ```
5. **同步 Obsidian 笔记**：将你的 Obsidian 知识库文件夹中的所有 `.md` 文件复制或链接到 Quartz 项目的 `content/` 目录下：
   - 推荐方式：在 `content/` 中创建指向 Obsidian 知识库的符号链接（macOS/Linux: `ln -s /path/to/your/vault content`；Windows: `mklink /J content D:\path\to\vault`）。
   - 替代方式：直接复制文件 `cp -r /path/to/vault/* content/`（但每次更新后需要手动复制）。
6. **本地预览**：运行 `npx quartz build --serve`，在浏览器中打开 `http://localhost:8080` 查看效果。
7. **部署到 GitHub Pages**：运行 `npx quartz build`，然后将 `public/` 目录中的内容推送到 GitHub：
   ```bash
   npx quartz build
   git add .
   git commit -m "deploy site"
   git push origin main
   ```
8. **为什么这么做**：完全免费（GitHub Pages 提供免费托管），支持完整的 Markdown 渲染、代码高亮、搜索功能、甚至是 Obsidian 的 `[[wikilink]]` 转换。
9. **停止条件**：`https://your-username.github.io` 能正常访问，内容与 Obsidian 中一致。

### 方案三：使用 Digital Garden 插件（轻量免费方案）

1. 安装 Digital Garden 插件：设置 → 第三方插件 → 社区市场 → 搜索 "Digital Garden" → 安装并启用。
2. 在插件设置中配置 GitHub 仓库信息（需要先在 GitHub 上创建一个空仓库，例如 `my-notes`）。
3. 在 GitHub 上启用 GitHub Pages：仓库 Settings → Pages → Source 选择 "GitHub Actions"。
4. 回到 Obsidian，右键点击任意笔记 → "Digital Garden: Note Status" → 设置为 "Published"。
5. 打开命令面板（`Ctrl+P`），输入 "Digital Garden: Publish all notes"。
6. **为什么这么做**：Digital Garden 插件比 Quartz 配置更简单，不需要手动处理构建流程——它在 Obsidian 内部完成了所有操作。但可选的主题和配置项也比 Quartz 少。
7. **停止条件**：`https://your-username.github.io/my-notes` 能看到你已经发布的笔记。

## 常见问题排查

1. **症状：Obsidian Publish 发布后笔记显示 404** → **根源**：笔记使用了未发布的 `[[wikilink]]` 指向 URL 错误。**解决**：在 Publish 控制面板中勾选所有被引用的笔记；如果某篇笔记不想公开，使用 `[链接文字](目标笔记.md)` 格式替代 `[[目标笔记]]`。

2. **症状：Quartz 构建时报错 "Error: ENOTDIR"** → **根源**：`content/` 符号链接在 Windows 上创建方式不正确。**解决**：在 Windows 上使用 `mklink /J content D:\path\to\vault`（注意是 `/J` 目录链接，不是 `/D` 符号链接），或以管理员身份运行终端。

3. **症状：Quartz 部署到 GitHub Pages 后显示空白页** → **根源**：`baseUrl` 配置错误，资源文件路径不正确。**解决**：确认 `quartz.config.ts` 中 `baseUrl` 的值为 `"your-username.github.io"`（不含 `https://`），然后重新构建并部署。

4. **症状：Digital Garden 发布后格式错乱** → **根源**：笔记中包含 Obsidian 特有的 Markdown 扩展（如备注块 `>[!note]`、图表 ````mermaid`），这些语法在某些渲染器中不被支持。**解决**：检查 Digital Garden 插件文档中支持的特性列表，或在发布前将特殊语法替换为通用 Markdown。

5. **症状：搜索引擎搜索不到已发布的内容** → **根源**：搜索引擎尚未收录你的站点。**解决**：对于 Obsidian Publish，在设置中开启 "SEO" 选项；对于 Quartz，确认生成的 HTML 包含正确的 `<meta>` 标签，然后在 Google Search Console 中提交站点地图。

## 专家技巧

- **Publish 的权限控制技巧**：Obsidian Publish 支持为站点设置密码保护——在 Publish 设置中开启 "Require password to access" 并设置密码。这使得它非常适合发布内部文档或团队知识库。
- **Quartz 的自定义样式**：在 `quartz/styles/custom.scss` 中覆盖 CSS 变量即可快速改变站点外观。例如：`--primary: #7c3aed;` 将主题色改为紫色——无需修改任何组件代码。
- **选择性发布的策略**：即使在使用免费方案时，也不建议一次性发布整个知识库。最佳实践是：先发布 10–20 篇最成熟的笔记，获取反馈后再逐步扩展。一篇高质量的笔记比 100 篇未整理的笔记更有价值。
- **更新频率**：Quartz 提供了 `npx quartz update` 命令来检查并合并上游更新。建议每月运行一次，避免 Quartz 版本落后导致的构建问题。

## 总结

你学会了三种将 Obsidian 笔记发布为网站的方法：Obsidian Publish（最快、付费）、Quartz + GitHub Pages（最灵活、免费）、Digital Garden 插件（最轻量、免费）。选择哪种方案取决于你的预算和技术水平——如果追求零配置，选 Publish；如果掌握命令行且想要完全控制，选 Quartz；如果想快速尝试免费方案，选 Digital Garden。

发布只是开始。要让你的站点长期有价值，请定期更新笔记、关注读者反馈、不断迭代内容。接下来可以阅读 [Obsidian 插件推荐与工作流](/zh/productivity/obsidian/plugins-workflow)，探索如何用 Dataview 和 Templater 自动化你的写作与发布流程。
