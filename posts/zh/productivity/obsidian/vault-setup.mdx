> **适用版本**：Obsidian 1.8.x（桌面端 v1.8.0+，移动端 v1.6.0+）
> **难度等级**：★☆☆☆☆（无需任何编程或知识管理经验）
> **预计耗时**：30–45 分钟

## 为什么你需要的不是另一个笔记应用

你可能尝试过很多笔记工具：在 Notion 里搭过复杂的数据库，用 Bear 写过零散的随想，或者在 Evernote 里堆了几千条再也没看过的剪藏。每次都以"这次一定要整理好"开始，三个月后面对一片混乱的文件夹结束。

问题不在你，而在于这些工具让你在还没理解自己如何思考之前，就强迫你设计一套分类体系。文件夹套文件夹、层层嵌套——等你想调整结构时，几百条笔记已经锁死在错误的树状目录里。

Obsidian 换了一个思路：不预设任何结构，从空白文件夹开始，用 Markdown 自由书写，通过 `[[wikilinks]]` 和标签让连接自然生长。你不需要"设计"知识库——它会在使用中自己长出来。六个月后，你会拥有一张映射你真实思维模式的知识图谱，而不是一个在第一天设计好、第二周就过时的文件夹架构。

本教程将带你从下载 Obsidian 开始，一步步搭建一个可持续使用、真正属于你的知识管理系统。

## 准备工作

开始之前，确保以下条件已满足：

- **软件**：从 [obsidian.md](https://obsidian.md) 下载安装 Obsidian 1.8+。桌面端支持 Windows 10+、macOS 12+、Linux（AppImage/Snap）。移动端支持 iOS 16+ 和 Android 10+。
- **账号**：不需要！Obsidian 所有数据默认存储在本地 `.md` 文件中，不依赖任何云端账号。可选：注册免费 Obsidian 账号，日后用于同步或发布功能。
- **操作系统技能**：了解基本的文件管理操作（新建文件夹、复制粘贴）。
- **建议先阅读**：无。本教程从零开始。

## 核心工作流

### 第一步：创建知识库

1. 首次启动 Obsidian，点击启动页中的"创建新知识库"。
2. 输入知识库名称，例如"我的知识库"。
3. 选择存放位置。建议放在云端同步文件夹内（如 iCloud Drive、Dropbox、OneDrive），这样自动跨设备同步。例如选择 `~/iCloudDrive/Obsidian/`。
4. 点击"创建"。
5. **预期结果**：编辑器打开，显示一篇名为"欢迎使用 Obsidian"的空白笔记。左侧是文件列表，右侧是当前笔记内容。
6. **为什么这么做**：每个知识库就是一个独立文件夹。你可以随时把它复制到 U盘、压缩备份、或通过网盘同步。不依赖任何特定服务商。

### 第二步：建立文件夹骨架

1. 在左侧文件列表中，右键 → "新建文件夹"。
2. 依次创建以下文件夹：`01-收集箱`、`02-项目`、`03-领域`、`04-资源`、`05-归档`。
3. **为什么这么做**：这套是 PARА 方法（Project-Areas-Resources-Archive）的变体，按行动状态而非主题分类。传统按主题分类（学习/工作/生活）的问题在于一条笔记往往属于多个主题——一篇"Python 爬虫"笔记同时是"编程"也是"工作项目"。
4. **预期结果**：文件列表显示五个空文件夹。前期大部分笔记会放在"收集箱"中。
5. **停止条件**：能在文件列表中看到这五个有序的文件夹。

### 第三步：写第一篇笔记

1. 点击左侧"新建笔记"按钮，或按快捷键 `Ctrl+N`（Mac 为 `Cmd+N`）。
2. 将标题改为"我的第一篇笔记"。
3. 在编辑器中输入以下内容体验 Markdown 语法：
   ```
   # 我的第一篇笔记

   这是一段 **加粗文字** 和 *斜体文字*。

   ## 子标题

   - 列表项一
   - 列表项二
   ```
4. **为什么这么做**：Obsidian 使用纯文本 Markdown 格式——这意味着你的笔记永远不会有"格式过期"或"导入导出乱码"的问题。即使十年后 Obsidian 不再更新，任何文本编辑器都能打开这些文件。
5. **停止条件**：你能在预览模式下看到格式化后的标题、粗体、列表。

### 第四步：创建笔记之间的链接

1. 选中一段文字（例如"第一篇笔记"），按 `[[` 键。
2. 在弹出的搜索框中输入笔记名称，然后回车。
3. Obsidian 会自动创建形如 `[[我的第一篇笔记]]` 的链接。
4. 再新建 2-3 篇笔记，相互之间建立链接。
5. 查看右侧的"反向链接"面板——它会列出所有链接到当前笔记的其他笔记。
6. **为什么这么做**：链接是 Obsidian 知识图谱的基石。每一条 `[[链接]]` 都是一次思维连接——让你从一个想法自然地跳到另一个相关想法。与传统文件夹不同，一条笔记可以被任意多篇笔记引用。
7. **停止条件**：你创建了至少 3 篇笔记，每篇包含 2 个以上的双向链接。

### 第五步：添加标签

1. 在一篇笔记的任意位置输入 `#` 加标签名，例如 `#学习`、`#项目规划`。
2. 标签会自动补全——你只需输入一次，后续输入相同标签时 Obsidian 会提示。
3. 在右侧边栏打开"标签列表"面板（快捷键 `Ctrl+Shift+7`），可以看到所有标签的汇总。
4. **为什么这么做**：文件夹是树状的——一条笔记只能在一个文件夹里。标签是网状的——一条笔记可以有多个标签。它们互相补充：文件夹负责粗粒度的分类，标签负责细粒度的检索。
5. **停止条件**：你给 3 篇笔记添加了 2–3 个不同的标签，并且能在标签面板中看到它们。

### 第六步：探索图谱视图

1. 点击左侧功能区的图谱图标，或按快捷键 `Ctrl+Shift+G`（Mac 为 `Cmd+Shift+G`）。
2. 图谱中每个圆点代表一篇笔记，每条线代表一个链接。
3. 用鼠标拖拽可以旋转 3D 视图，滚动滚轮缩放。
4. 点击任意节点，图谱会高亮该节点及所有连接节点。
5. **为什么这么做**：图谱视图让你一眼看到知识的全貌——哪些区域连接紧密（说明这些笔记相互关联），哪些节点孤立（提示需要补充链接或归档）。
6. **停止条件**：你能看到至少 5 个相互连接的节点网络。

## 常见问题排查

1. **症状：我在另一台电脑上看不到笔记** → **根源**：Obsidian 默认将数据存储在本地，不提供自动云同步。**解决**：将知识库文件夹放在 iCloud、Dropbox 或 OneDrive 目录中，或在多台电脑之间使用 Obsidian Sync 官方同步服务。

2. **症状：我的 wikilink 显示为灰色/未解析** → **根源**：链接的目标笔记不存在，或者文件名已更改。**解决**：点击灰色链接会自动创建对应的新笔记；如果是更名导致，用 `Ctrl+O` 找到原文件后重命名。

3. **症状：图谱视图里只有一个孤零零的节点** → **根源**：你还没有在任何笔记之间建立链接。**解决**：在笔记中添加 `[[其他笔记名]]`，图谱只会显示有连接的节点。

4. **症状：编辑器显示原始 Markdown 代码，而不是格式化文本** → **根源**：当前处于"源码模式"，而非"实时预览"。**解决**：按 `Ctrl+E` 切换到实时预览模式，或在设置 → 编辑器 → 默认视图模式中修改。

5. **症状：文件列表里出现了大量 `.DS_Store` 或 `Thumbs.db` 文件** → **根源**：操作系统生成的隐藏文件。**解决**：设置 → 文件与链接 → 排除的文件模式，添加 `.*` 和 `Thumbs.db` 即可隐藏。

## 专家技巧

- **原子笔记原则**：每篇笔记只写一个概念。如果一篇笔记超过 300 字或涵盖多个主题，考虑拆分。笔记越短小、越聚焦，搜索和链接的效率越高。在实际使用中，一篇 50–100 字的笔记往往比长篇大论更有价值。
- **PARA 文件夹使用比例**：在持续使用 3 个月后，约 40% 的笔记会进入"归档"文件夹。这很正常——归档不是"废弃"的意思，而是"当前不活跃但有参考价值"。如果"归档"占比低于 20%，说明你不敢清理——这会降低"项目"和"领域"文件夹的质量。
- **日记是收集箱的前端**：启用核心插件"日记"（设置 → 核心插件 → 日记），按 `Ctrl+Shift+D` 打开今日日记。日常工作中约 70% 的碎片想法先用日记随手记录，然后在每周回顾中整理到相应的笔记中。这比随时停下来整理要高效得多。
- **使用别名处理同义词**：当一篇笔记有多个名称时（例如"AI"和"人工智能"），在笔记的开头添加 `aliases: [AI, Artificial Intelligence]` 并开启"属性"核心插件。这样无论你用哪个名称搜索或链接，都能找到同一篇笔记。
