## 静态稿说服不了任何人

你有一份精美的 UI 设计稿，但向客户演示时，你必须不断口头解释："点击这里会弹出一个菜单""这个页面会滑动切换"——客户点头，但眼神里写满了不确定。根据 Nielsen Norman Group 的研究，**用户与高保真可交互原型的互动** 能提前发现 85% 的可用性问题，而静态稿只能发现 30%。

Figma 的原型功能让你无需编写任何代码，就能把静态设计变成可点击、可滑动、带过渡动画的交互体验。本教程将带你从连接基础交互到高级条件逻辑，完整覆盖原型设计流程。

## 准备工作

- **软件**: Figma（免费版支持基本原型功能，条件逻辑需要 Figma Professional 及以上）

- **前置知识**: 熟悉 Figma 页面管理（Page 面板）、组件基础、自动布局

## 核心工作流

### 第一步：建立页面流程

在开始连接交互之前，先规划用户流程：

1. 打开包含多页面的设计文件（如：登录页 → 首页 → 个人中心）
2. 确保每页的主要交互元素都有明确的图层命名（如 `btn_login`、`card_product`）
3. 切换到右上角的 **原型** 标签页（Prototype tab）

**为什么这样做？** 清晰的图层命名直接影响原型连接的效率——在连线模式下，你通过悬停图层名称来定位目标，杂乱命名将浪费大量时间。

**停止条件**: 画布右上角从"设计"切换到"原型"模式，右侧属性面板变为原型交互设置面板。

### 第二步：创建基础页面跳转（点击交互）

最简单的交互——点击按钮跳转到另一页面：

1. 选中登录页上的登录按钮（`btn_login`）
2. 点击右侧原型面板中的 **+** 号（或直接拖拽圆圈手柄到目标页面）
3. 在交互设置面板中：
   - **触发动作**: 点击（On Click）
   - **动作**: 导航到（Navigate To）
   - **目标**: 选择"首页"页面
   - **过渡动画**: Smart Animate
   - **时长**: 300ms
   - **缓动曲线**: Ease Out

**为什么这样做？** Smart Animate 会自动识别两个页面中相同名称的图层，并在过渡时生成平滑的移动/缩放/透明动画。300ms 是 material design 推荐的交互时长——太短显得生硬，太长让用户等待。

**停止条件**: 点击右上角的播放 ▶ 按钮进入演示模式，点击登录按钮，页面平滑过渡到首页。

### 第三步：叠加层——弹窗和菜单

页面跳转并不适用于所有场景。弹窗、菜单、提示框应该使用叠加层：

1. 在首页上画一个 320×auto 的底部菜单面板
2. 选中触发元素（如"分享"图标）
3. 创建新交互：
   - **触发动作**: 点击
   - **动作**: 打开叠加层（Open Overlay）
   - **目标**: 选择刚才画的底部菜单
   - **叠加层位置**: 底部居中（Bottom Center）
   - **背景**: 勾选"背景模糊"（Background Blur），设置强度 15px

**为什么这样做？** 叠加层不改变页面上下文，用户关闭后仍停留在原页面。背景模糊让用户知道"后面还有内容"，同时聚焦到弹窗上。底部菜单定位确保在移动设备上使用体验自然。

**停止条件**: 播放演示，点击分享图标，底部菜单从下方向上滑入，背景虚化。

### 第四步：滚动内容与固定元素

长列表页面需要正确的滚动行为：

1. 在首页创建一个长内容区域（超出画布高度，例如 800px 高）
2. 选中该内容区域，在右侧原型面板找到 **滚动** 设置
3. 设置：
   - **滚动方向**: 垂直（Vertical）
   - **适配**: 内容高度（Fit to content height）
4. 选中顶部导航栏，设置为 **固定位置**（Stick to top）

**为什么这样做？** 不设置滚动时，整个页面会一起滚动，导航栏会滚出视野。固定导航栏模拟了真实的 App 体验，让用户在查看长内容时始终知道当前页面。

**停止条件**: 播放演示，页面内容可上下滚动，导航栏固定在顶部不动。

### 第五步：动效曲线与 Smart Animate 进阶

默认的 Ease Out 是不够的。不同的 UI 元素需要不同的动效语言：

1. 创建两个卡片页面之间切换的交互
2. 在交互设置中尝试不同的缓动曲线：
   - **Ease Out**: 快速开始，缓慢结束（适合页面过渡，$300ms$）
   - **Ease In Out**: 两端缓慢（适合弹窗，$400ms$）
   - **Spring**: 弹性效果，设置 tension 400, friction 30（适合卡片翻转，$500ms$）
3. 对比同一页面使用不同曲线的效果

**为什么这样做？** Ease Out 让人感觉界面响应迅速，因为它在前 100ms 内完成了大部分位移。Spring 效果传递"物理感"，适合需要强调的交互。Google Material Motion 指南建议不同类型的元素使用不同的时间尺度。

**停止条件**: 同一交互使用三种动画方案后，Spring 版本有明显的弹性回弹效果。

### 第六步：条件逻辑——根据状态展示不同页面

这是 Figma Professional 的高级功能，让你的原型真正"智能"：

1. 创建两个用户状态：登录状态和未登录状态
2. 在首页的"个人中心"入口图标上创建交互：
   - **触发动作**: 点击
   - **动作**: 条件判断（Conditional）
3. 设置条件：
   - 如果 `userLoggedIn` = true → 导航到"个人中心"
   - 如果 `userLoggedIn` = false → 打开"登录弹窗"
4. 在顶部创建 **变量**：点击右侧面板的 Variables 标签，创建布尔变量 `userLoggedIn`，初始值设为 false

**为什么这样做？** 条件逻辑让你的原型模拟真实的业务逻辑，而不用为每个状态单独制作原型。在用户测试时，条件逻辑展示的真实感远高于手动切换页面。

**停止条件**: 演示模式下，首次点击个人中心跳转到登录页；通过在变量面板中把 `userLoggedIn` 改为 true，再次点击进入个人中心。

### 第七步：原型录制与分享

原型完成后，可以录制成视频分享给团队：

1. 点击右上角 **分享原型**（Share prototype）
2. 在分享对话框设置：
   - **权限**: "任何有链接的人可查看"
   - **密码保护**: 可选
   - **到期时间**: 可选
3. 勾选 **录制**（Record prototype）——Figma 会录制你的操作轨迹
4. 分享链接给利益相关者，他们可以在浏览器中直接点击交互

**为什么这样做？** 相较于导出 MP4 视频，分享在线原型链接的优点是：接收者可以自己点击探索，而不是被动观看。Figma 的录制功能也能自动生成操作指南。

**停止条件**: 打开分享链接（或切换为隐身模式测试），页面加载后可以像真实 App 一样点击操作。

## 常见问题

### Smart Animate 没有产生平滑过渡

**原因**: Smart Animate 要求两个页面的目标图层**名称完全一致**。如果按钮在 A 页叫 `btn_next`，在 B 页叫 `btn_submit`，则无法匹配。
**解决**: 进入需要过渡的两个页面，确保参与动画的图层名称一致（大小写区分）。Figma 忽略图层在层级中的位置——只按名称匹配。如果图层结构复杂，建议使用统一的命名规范。

### 叠加层关闭后页面内容消失了

**原因**: 叠加层的背景模糊设置导致页面内容被错误地处理为"背景层"，关闭时触发了页面卸载。
**解决**: 不要在主页面中使用叠加层作为全屏内容容器。如果必须使用全屏遮罩，考虑使用页面跳转 + 返回代替。或者取消勾选叠加层设置中的"Close overlay"自动行为，使用手动关闭逻辑。

### 滚动设置后页面无法滚动

**原因**: 最常见的错误是把滚动设置在 Page 上而不是 Frame 上。Figma 的原型滚动必须挂在 Frame（容器）上，Page 本身不支持滚动。
**解决**: 确保长内容被包裹在一个 Frame 中（而不是直接放在 Page 上）。选中该 Frame → Prototype 属性 → 设置 Scroll behavior 为 Vertical。Frame 的高度设为 Fit to content。

### 条件逻辑中的变量没有生效

**原因**: 变量作用域问题——你在一个页面集合中定义了变量，但交互目标不在同一集合内。
**解决**: 点击 Variables 标签页，检查变量的 Scope（作用域）设置。如果交互跨越不同的 Flow（流程），需要将变量设为全局（Global）。变量名不区分大小写，但建议用小驼峰命名（如 `isLoggedIn`）以保持一致性。

## 专家技巧

- **交互预览快捷键**: 在演示模式下按 `Ctrl+\` / `Cmd+\` 显示所有可交互区域的蓝色覆盖层，快速检查是否有漏连的交互点
- **多起点原型**: 一个文件可以创建多个 Flow 起点。配合组件变体，一个文件可以同时演示登录流程、注册流程和设置流程
- **Spring 参数调优**: Tension 控制弹性刚度（越大回弹越快），Friction 控制阻尼（越大停止越稳）。推荐配对：弹窗显现用 tension 300/friction 25，卡片切换用 tension 400/friction 30
- **演示模式快捷键**: 按 `1-9` 快速缩放，按 `R` 重新开始，按 `F` 切换全屏。这些在不使用鼠标时非常有用——特别是在用户测试场景
- **原型性能**: 单个原型中的交互连接超过 200 个时，演示模式可能出现延迟。**使用组件 + 变体**而非每个页面单独画元素，能显著减少连接数
