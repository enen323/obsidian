## 还在手动复制粘贴设计稿？

你做过这样的事情吗：花了一周设计了 30 个页面，客户突然说"按钮颜色改成品牌蓝"。于是你开始一页一页地改——改到第 12 页时发现漏了一页，改完又发现按钮阴影参数不一致。根据 Figma 官方 2023 年的设计工具调查，**67% 的设计师**认为"跨页面一致性维护"是团队协作中最大的痛点。

**组件系统**就是为此而生。它让你创建一个"母版"按钮，所有副本自动跟随更新。本教程将带你从零搭建完整的组件库——从基础组件到变体，从样式管理到团队库发布。

## 准备工作

- **软件**: Figma（浏览器版免费注册，建议使用桌面客户端以获得更好的字体渲染）
- **账号**: 无需付费账号，免费版即可完成本教程全部操作。如果使用团队库功能，需要 Figma Professional 及以上方案
- **文件**: 无需现有文件，我们将从头创建
- **前置知识**: 了解 Figma 的基本界面（图层面板、属性面板），会使用基本形状工具

## 核心工作流

### 第一步：创建基础按钮组件

打开 Figma，新建文件（`Ctrl+N` / `Cmd+N`）。

1. 用矩形工具（`R`）绘制一个 120×40 的矩形，圆角 6px
2. 填充颜色设为 `#3B82F6`（蓝色），去掉描边
3. 用文字工具（`T`）输入"提交"，字号 14px，字重 Medium，颜色白色
4. 选中矩形和文字，点击顶部工具栏的 **创建组件** 按钮（或按 `Ctrl+Alt+K` / `Cmd+Option+K`）

**为什么这样做？** 组件化后，所有副本共享同一个"主组件"。修改主组件，所有实例自动更新。这意味着你不再需要逐个调整每个按钮。

**停止条件**: 你会在图层面板中看到组件以紫色边框标识，且组件名前有一个「◆」符号。

### 第二步：设置组件属性——让按钮可配置

目前按钮文字是固定的。我们需要让它可复用：

1. 选中刚才创建的组件，在右侧属性面板找到 **属性** 区域
2. 点击文字图层的"文本"属性旁的 **+** 号，将其公开为组件属性
3. 将属性名称改为 `label`，默认值设为"提交"
4. 同样，将矩形的填充颜色公开为属性，命名为 `bgColor`

**为什么这样做？** 属性让组件的使用者在保持主组件控制权的同时，可以自定义实例的特定参数。这比"复制后直接改"更安全——不会被误操作破坏组件结构。

**停止条件**: 右侧 Properties 面板显示两个可编辑字段：`label`（文本输入）和 `bgColor`（颜色选择器）。

### 第三步：创建变体——一个组件，多种状态

一个按钮不可能只有一种状态。我们需要实现 Default、Hover、Pressed、Disabled 四种状态：

1. 选中主组件，按住 `Alt` 向右拖动复制一个
2. 在新副本中，将矩形颜色改为 `#2563EB`（深蓝）——这将是 Hover 状态
3. 再复制两个：Pressed 用 `#1D4ED8`，Disabled 用 `#9CA3AF`（灰色）
4. 选中全部四个组件，点击属性面板中的 **组合为变体** 按钮（或右侧属性面板的「Combine as variants」）

**为什么这样做？** 变体把多个视觉状态打包到同一个组件名下，通过属性切换。开发者拿到的是一个组件、四种状态，而不是四个独立的组件。这在设计交接时能大幅减少沟通成本。

**停止条件**: 你会看到四个组件合并为一个，右侧属性面板出现一个下拉选择器，可在四个状态间切换。

### 第四步：定义变体属性

1. 选中变体组，在右侧属性面板中，把默认的 `Property 1` 重命名为 `state`
2. 将四个值分别命名为：`default`、`hover`、`pressed`、`disabled`
3. 点击 `disabled` 值旁的编辑按钮，设置交互能力：取消勾选"可点击"，因为禁用态不应触发点击

**为什么这样做？** 命名清晰的变体属性让团队一目了然。开发人员看到 `state=disabled` 马上知道这是禁用状态，不需要再打开组件内部查看。

**停止条件**: 在画布上复制一个变体实例，右侧属性面板中你可以通过下拉菜单在四种状态间切换。

### 第五步：创建样式——颜色与文字规范

组件需要统一的样式标准，不能每次手动选颜色：

1. 选中蓝色矩形，点击右侧属性面板中填充旁的 **样式** 图标（四个点组成的菱形）
2. 点击 **创建样式** → 选择 **颜色样式**
3. 命名为 `Brand/Blue/500`，发布范围设为"当前文件"
4. 同样，其他几种蓝色分别保存为 `Brand/Blue/600`、`Brand/Blue/700`，灰色保存为 `Neutral/Gray/300`
5. 用文字工具创建一个 14px Medium 的文字样式，保存为 `Body/14/Medium`

**为什么这样做？** 颜色样式的层级命名（`品牌/蓝色/500`）让你在不同项目中保持一致性。改动 `Brand/Blue/500` 的定义，所有使用该样式的组件自动更新。

**停止条件**: 在组件中右键点击填充或文字，可以看到已保存的样式出现在快捷菜单中。

### 第六步：使用自动布局让组件自适应

现在我们的按钮宽度是固定的 120px。但如果文字变成"确认提交"？按钮需要自适应：

1. 选中组件内的所有图层（矩形 + 文字）
2. 按 `Shift+A` 添加 **自动布局**
3. 在右侧属性面板中设置：
   - 方向：水平
   - 间距：8px（图标和文字之间）
   - 内边距：上下 10px，左右 16px
   - 水平对齐方式：居中
4. 选中矩形，在右侧将矩形宽度设为 **Fill**（填充容器），高度设为 **Fill**

**为什么这样做？** 自动布局让组件像网页 Flexbox 一样工作。文字变长时，按钮自动延展；添加图标时，间距自动保持 8px。不再需要手动调整每个实例。

**停止条件**: 尝试在实例中把文字改为"确认提交"，按钮宽度自动增加，文字始终居中。

### 第七步：发布团队库

如果你在团队中工作，需要把组件共享出去：

1. 点击图层面板上方的 **资源** 标签页
2. 点击 **发布库** 按钮
3. 选择你要发布的样式和组件
4. 填写版本说明，例如"v1.0 - 基础按钮组件 + 品牌色板"
5. 点击 **发布**
6. 团队成员在「资源」面板中启用该库即可使用

**为什么这样做？** 团队库让所有成员使用同一套组件。新人加入项目时不用问"咱们的按钮在哪里"——库会自动推送到他们的 Figma 中。

**停止条件**: 在另一个文件中，点击「资源」→「团队库」，勾选你刚发布的库，你会看到组件出现在左侧面板中。

## 常见问题

### 修改了主组件，但实例没有更新

**原因**: 最可能是在实例上直接修改了某个属性（比如改了颜色），局部覆盖会阻止实例跟随主组件更新。
**解决**: 右键该实例 → **Reset overrides**（重置覆盖）。如果只想重置特定属性，在右侧属性面板中点击该属性旁的撤销图标。

### 变体下拉菜单中看不到我创建的状态

**原因**: 你在"变体"组外创建了新组件，没有正确合并进来。
**解决**: 选中新组件和现有的变体组，确保它们在同一个 Frame 中，然后点击属性面板的「Combine as variants」。注意：所有变体必须处于同一个自动布局或 Frame 内。

### 团队库发布后成员看不到

**原因**: 成员没有启用该库，或者发布时没有选择正确的团队空间。
**解决**: 让成员打开文件，点击「资源」面板 → 「团队库」标签，手动开关对应库。如果库未出现，检查你是否发布到了正确的团队——发布对话框中会显示目标团队名称。

### 自动布局中的组件间距不一致

**原因**: 不同实例的文本长度不同，但你没有设置最小宽度或最大宽度约束。
**解决**: 在组件主版中，选中自动布局容器，在右侧属性面板设置最小宽度（Min Width）为 68px（保证短文字按钮不会缩得太小），最大宽度（Max Width）设置为 320px（防止极长文字撑爆布局）。

## 专家技巧

- **批量替换组件**: 选中所有需要替换的实例 → 在右侧属性面板点击组件名称 → 从下拉菜单中选择目标组件。Figma 会自动保留你所做过的文本和颜色覆盖。熟练使用后，一次 50+ 实例的替换只需 10 秒。
- **嵌套组件策略**: 一个图标按钮可以嵌套在更大的组件中。典型场景：卡片组件内部嵌入按钮组件。嵌套深度建议不超过 3 层，否则维护成本会快速上升。
- **变体属性排序**: 在变体属性面板中，拖拽属性名称左侧的六个点可以调整排列顺序。把最常用的属性放在最上面，减少鼠标滚动。
- **组件命名规范**: 推荐 `类型/名称/状态` 格式，例如 `Button/Primary/Default`。Figma 的资源面板会按名称的 `/` 自动分组，这比扁平命名清晰得多。
- **性能优化**: 单个 Figma 文件中组件数量超过 500 个时，文件性能可能下降。此时建议拆分为多个文件，通过团队库互相引用。
